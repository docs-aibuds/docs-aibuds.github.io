Log Viewing

This guide will walk you through how to view and analyze logs generated by the AIBuds SDK.

View Logs in Console​

You can view logs directly in the Xcode console during development:

Objective-C​

// Enable console logging
[AIBudsLogManager.shared setConsoleLoggingEnabled:YES];

// Set log level to debug for detailed information
[AIBudsLog setLogLevel:AIBudsLogLevelDebug];

Swift​

// Enable console logging
AIBudsLogManager.shared.setConsoleLoggingEnabled(true)

// Set log level to debug for detailed information
AIBudsLog.setLogLevel(.debug)

View Log Files​

The AIBuds SDK can save logs to files on the device. You can access these log files in several ways:

Using the Files App​

  1. Enable file logging in your app
  2. Run your app to generate logs
  3. Open the Files app on your device
  4. Navigate to "On My iPhone" > Your App > Logs
  5. Tap on log files to view them

Using Xcode​

  1. Connect your device to your Mac
  2. Open Xcode and go to Window > Devices and Simulators
  3. Select your device and click "View Device Logs"
  4. Filter logs by your app name

Using ADB (Android Debug Bridge)​

For iOS devices, you can use the ideviceconsole tool to view logs:

ideviceconsole -u <device-udid> -e <bundle-id>

Log File Structure​

Log files are organized by date and time, with each log file containing logs for a specific period:

Logs/
├── 2023-06-01_10-00-00.log
├── 2023-06-01_11-00-00.log
└── 2023-06-01_12-00-00.log

Each log entry follows the configured log format, typically including:

  • Date and time
  • Log level
  • Category
  • File and line number
  • Thread information
  • Log message

Log Viewer​

The AIBuds SDK includes a built-in log viewer that you can integrate into your app:

Objective-C​

// Present log viewer
[AIBudsLogViewer presentLogViewerFromViewController:self];

Swift​

// Present log viewer
AIBudsLogViewer.presentLogViewer(from: self)

Export Logs​

You can export logs from the device for analysis:

Objective-C​

// Export logs
[logManager exportLogsWithCompletion:^(NSURL *fileURL, NSError *error) {
    if (fileURL) {
        NSLog(@"Logs exported to: %@", fileURL.path);
        
        // Share the log file
        UIActivityViewController *activityVC = [[UIActivityViewController alloc] initWithActivityItems:@[fileURL] applicationActivities:nil];
        [self presentViewController:activityVC animated:YES completion:nil];
    } else {
        NSLog(@"Failed to export logs: %@", error.localizedDescription);
    }
}];

Swift​

// Export logs
logManager.exportLogs { fileURL, error in
    if let fileURL = fileURL {
        print("Logs exported to: \(fileURL.path)")
        
        // Share the log file
        let activityVC = UIActivityViewController(activityItems: [fileURL], applicationActivities: nil)
        self.present(activityVC, animated: true, completion: nil)
    } else {
        print("Failed to export logs: \(error?.localizedDescription ?? "Unknown error")")
    }
}

Log Analysis​

When analyzing logs, look for the following:

Error Messages​

Error messages indicate issues that need attention. They typically start with [ERROR] and include details about what went wrong.

Warning Messages​

Warning messages indicate potential issues that may not be critical but should be addressed. They typically start with [WARNING].

Performance Issues​

Look for performance-related logs that indicate slow operations or high resource usage.

Bluetooth Issues​

Logs related to Bluetooth connectivity can help diagnose connection problems:

[DEBUG] [Bluetooth] Scanning for devices
[INFO] [Bluetooth] Found device: AIBuds Pro
[ERROR] [Bluetooth] Failed to connect to device: Connection timeout

Audio Issues​

Logs related to audio processing can help diagnose audio problems:

[DEBUG] [Audio] Starting audio stream
[WARNING] [Audio] Audio buffer underrun detected
[INFO] [Audio] Stopping audio stream

Log Filtering​

You can filter logs to focus on specific areas:

Objective-C​

// Filter logs by category
[logManager setLoggingEnabled:YES forCategory:@"Bluetooth"];
[logManager setLoggingEnabled:YES forCategory:@"Audio"];
[logManager setLoggingEnabled:NO forCategory:@"AI"];

// Filter logs by level
[AIBudsLog setLogLevel:AIBudsLogLevelError];

Swift​

// Filter logs by category
logManager.setLoggingEnabled(true, forCategory: "Bluetooth")
logManager.setLoggingEnabled(true, forCategory: "Audio")
logManager.setLoggingEnabled(false, forCategory: "AI")

// Filter logs by level
AIBudsLog.setLogLevel(.error)

Remote Logging​

If you've enabled remote logging, you can view logs on your remote server:

  1. Log in to your logging dashboard
  2. Select your app
  3. View logs by date, time, device, or severity
  4. Search for specific log messages or patterns
  5. Set up alerts for critical errors

Logging Best Practices​

For Debugging​

  1. Use detailed log levels during development
  2. Include context information to help identify issues
  3. Log important events and state changes
  4. Use consistent log formats
  5. Export and share logs when reporting issues

For Production​

  1. Use appropriate log levels to balance detail with performance
  2. Enable file logging to capture errors
  3. Consider remote logging for crash and error reporting
  4. Rotate log files to prevent excessive storage usage
  5. Obfuscate sensitive information in logs
