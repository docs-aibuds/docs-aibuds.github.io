Logging Overview

Welcome to the Logging documentation for the AIBuds SDK. This section covers APIs for managing logging functionality, including log output configuration, export, and cleanup.

Available Features​

📄️Output Methods

Configure logging output methods

📄️Export Logs

Export device logs to a file

📄️Delete Expired Logs

Delete expired log files from the device

Getting Started​

To use logging features, follow these steps:

  1. Initialize Log SDK - Set up the AIBudsLog SDK
  2. Configure Output - Select log output destination (console, file, etc.)
  3. Export Logs - Export logs for debugging
  4. Cleanup Logs - Delete expired log files

Key Concepts​

Log Destinations​

  • Console: Output logs to console
  • OS Logger: Use system logging framework
  • File: Write logs to file
  • XL Facility: Send logs to XL facility service

Log Management​

  • Export: Export logs to single or multiple files
  • Purge: Delete logs older than specified days
  • Configuration: Set log file size limits and retention policies

SDK Initialization​

The logging functionality is accessed through the AIBudsLogSDK:

  • Swift
  • Objective-C
import AIBudsLog

// Configure log destination
AIBudsLogConfiguration.shared.destination = .file

// Export logs
AIBudsLogSDK.logService.exportLogs { filePath, startDate, endDate, error in
    if let path = filePath {
        print("Logs exported to: \(path)")
    }
}
#import <AIBudsLog/AIBudsLog.h>

// Configure log destination
[AIBudsLogConfiguration shared].destination = AIBudsLogDestinationFile;

// Export logs
[AIBudsLogSDK.logService exportLogsWithCompletion:^(NSString *filePath, NSDate *startDate, NSDate *endDate, NSError *error) {
    if (filePath) {
        NSLog(@"Logs exported to: %@", filePath);
    }
}];

Best Practices​

  1. Configure Early: Set up logging configuration early in your app lifecycle
  2. Select Appropriate Destination: Choose log destination based on environment
  3. Manage Log Files: Regularly purge old logs to manage storage
  4. Handle Exports: Provide users with options to export logs for support

Notes​

  • Log configuration affects where logs are stored
  • Log file size limits help manage storage usage
  • Log exports are useful for debugging and support
