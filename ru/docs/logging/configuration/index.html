Logging Configuration

This guide will walk you through the logging configuration options in the AIBuds SDK.

Log Manager​

The AIBudsLogManager is the main class for logging functionality:

Objective-C​

// Get the log manager singleton
AIBudsLogManager *logManager = AIBudsLogManager.shared;

Swift​

// Get the log manager singleton
let logManager = AIBudsLogManager.shared

Set Log Level​

You can set the log level to control the amount of logging output:

Objective-C​

// Set log level
[AIBudsLog setLogLevel:AIBudsLogLevelDebug];

// Available log levels:
// AIBudsLogLevelNone - No logging
// AIBudsLogLevelError - Only error messages
// AIBudsLogLevelWarning - Error and warning messages
// AIBudsLogLevelInfo - Error, warning, and info messages
// AIBudsLogLevelDebug - All messages (including debug)
// AIBudsLogLevelVerbose - All messages including verbose debug information

Swift​

// Set log level
AIBudsLog.setLogLevel(.debug)

// Available log levels:
// .none - No logging
// .error - Only error messages
// .warning - Error and warning messages
// .info - Error, warning, and info messages
// .debug - All messages (including debug)
// .verbose - All messages including verbose debug information

Log Output Configuration​

You can configure where log messages are output:

Objective-C​

// Enable console logging
[logManager setConsoleLoggingEnabled:YES];

// Enable file logging
[logManager setFileLoggingEnabled:YES];

// Set log file directory
NSString *logDirectory = [NSSearchPathForDirectoriesInDomains(NSDocumentDirectory, NSUserDomainMask, YES) firstObject];
[logManager setLogDirectory:logDirectory];

// Set log file maximum size (in bytes)
[logManager setLogFileMaxSize:1024 * 1024 * 10]; // 10 MB

// Set maximum number of log files
[logManager setMaxLogFiles:5];

Swift​

// Enable console logging
logManager.setConsoleLoggingEnabled(true)

// Enable file logging
logManager.setFileLoggingEnabled(true)

// Set log file directory
let logDirectory = NSSearchPathForDirectoriesInDomains(.documentDirectory, .userDomainMask, true)[0]
logManager.setLogDirectory(logDirectory)

// Set log file maximum size (in bytes)
logManager.setLogFileMaxSize(1024 * 1024 * 10) // 10 MB

// Set maximum number of log files
logManager.setMaxLogFiles(5)

Log Format Configuration​

You can configure the format of log messages:

Objective-C​

// Set log format
[logManager setLogFormat:AIBudsLogFormatDetailed];

// Available log formats:
// AIBudsLogFormatSimple - Simple format: [LEVEL] Message
// AIBudsLogFormatStandard - Standard format: [DATE TIME] [LEVEL] [FILE:LINE] Message
// AIBudsLogFormatDetailed - Detailed format: [DATE TIME] [LEVEL] [FILE:LINE] [THREAD] Message

// Enable/disable color logging in console
[logManager setConsoleColorEnabled:YES];

// Set date format
[logManager setDateFormat:@"yyyy-MM-dd HH:mm:ss.SSS"];

Swift​

// Set log format
logManager.setLogFormat(.detailed)

// Available log formats:
// .simple - Simple format: [LEVEL] Message
// .standard - Standard format: [DATE TIME] [LEVEL] [FILE:LINE] Message
// .detailed - Detailed format: [DATE TIME] [LEVEL] [FILE:LINE] [THREAD] Message

// Enable/disable color logging in console
logManager.setConsoleColorEnabled(true)

// Set date format
logManager.setDateFormat("yyyy-MM-dd HH:mm:ss.SSS")

Category-based Logging​

You can enable or disable logging for specific categories:

Objective-C​

// Enable logging for specific category
[logManager setLoggingEnabled:YES forCategory:@"Bluetooth"];

// Disable logging for specific category
[logManager setLoggingEnabled:NO forCategory:@"Audio"];

// Get list of available categories
NSArray<NSString *> *categories = [logManager getAvailableLogCategories];
NSLog(@"Available log categories: %@", categories);

Swift​

// Enable logging for specific category
logManager.setLoggingEnabled(true, forCategory: "Bluetooth")

// Disable logging for specific category
logManager.setLoggingEnabled(false, forCategory: "Audio")

// Get list of available categories
let categories = logManager.getAvailableLogCategories()
print("Available log categories: \(categories)")

Logging Context​

You can add context information to log messages:

Objective-C​

// Add context information
[logManager addContextValue:@"1.0.0" forKey:@"appVersion"];
[logManager addContextValue:device.name forKey:@"deviceName"];

// Remove context information
[logManager removeContextValueForKey:@"appVersion"];

// Clear all context information
[logManager clearContext];

Swift​

// Add context information
logManager.addContextValue("1.0.0", forKey: "appVersion")
logManager.addContextValue(device.name, forKey: "deviceName")

// Remove context information
logManager.removeContextValue(forKey: "appVersion")

// Clear all context information
logManager.clearContext()

Custom Log Handlers​

You can add custom log handlers to process log messages:

Objective-C​

// Add custom log handler
[AIBudsLog addLogHandler:^(AIBudsLogLevel level, NSString *category, NSString *message, NSString *file, NSInteger line, NSString *function) {
    // Process log message
    NSLog(@"Custom log: [%@] %@", category, message);
}];

Swift​

// Add custom log handler
AIBudsLog.addLogHandler { level, category, message, file, line, function in
    // Process log message
    print("Custom log: [\(category)] \(message)")
}

Performance Logging​

You can enable performance logging to track the performance of SDK operations:

Objective-C​

// Enable performance logging
[logManager setPerformanceLoggingEnabled:YES];

// Set performance logging threshold (in milliseconds)
[logManager setPerformanceLoggingThreshold:10.0];

Swift​

// Enable performance logging
logManager.setPerformanceLoggingEnabled(true)

// Set performance logging threshold (in milliseconds)
logManager.setPerformanceLoggingThreshold(10.0)

Remote Logging​

You can enable remote logging to send log messages to a remote server:

Objective-C​

// Enable remote logging
[logManager setRemoteLoggingEnabled:YES];

// Set remote logging URL
[logManager setRemoteLoggingURL:[NSURL URLWithString:@"https://logs.example.com"]];

// Set remote logging interval (in seconds)
[logManager setRemoteLoggingInterval:30.0];

// Set remote logging batch size
[logManager setRemoteLoggingBatchSize:100];

Swift​

// Enable remote logging
logManager.setRemoteLoggingEnabled(true)

// Set remote logging URL
logManager.setRemoteLoggingURL(URL(string: "https://logs.example.com")!)

// Set remote logging interval (in seconds)
logManager.setRemoteLoggingInterval(30.0)

// Set remote logging batch size
logManager.setRemoteLoggingBatchSize(100)

Logging Best Practices​

For Development​

  • Use AIBudsLogLevelDebug or AIBudsLogLevelVerbose to get detailed logs
  • Enable console logging for immediate feedback
  • Enable file logging to review logs later

For Production​

  • Use AIBudsLogLevelWarning or AIBudsLogLevelError to reduce log volume
  • Disable console logging to avoid cluttering the device console
  • Enable file logging to capture errors for debugging
  • Consider enabling remote logging to collect crash and error information
