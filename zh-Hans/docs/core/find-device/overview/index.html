Find Device Overview

Welcome to the Find Device documentation for the AIBuds SDK. This section covers APIs for locating your AIBuds device using sound feedback, helping users find misplaced devices.

Available Features​

📄️查找设备

触发设备发出声音以便定位

📄️停止查找设备

阻止设备发出寻找声音

Getting Started​

To use the find device feature, follow these steps:

  1. Connect to Device - Establish a connection with your AIBuds device
  2. Check Protocol Conformance - Verify the device conforms to DeviceFindAPI protocol
  3. Initiate Find - Call the find device API to make the device emit sound

Key Concepts​

Find Device Operation​

  • Find: Triggers the device to emit a sound to help locate it
  • Stop Find: Stops the device from emitting sound

Protocol Reference​

The find device functionality is accessed through the DeviceFindAPI protocol:

  • Swift
  • Objective-C
guard let device = device as? DeviceFindAPI else {
    print("Device does not support DeviceFindAPI")
    return
}
id<AIBudsDeviceFindAPI> device = (id<AIBudsDeviceFindAPI>)self.device;
if (![device conformsToProtocol:@protocol(AIBudsDeviceFindAPI)]) {
    NSLog(@"Device does not support AIBudsDeviceFindAPI");
    return;
}

Best Practices​

  1. Check Protocol Conformance: Always verify the device supports the DeviceFindAPI protocol before calling methods
  2. Handle Errors: Implement proper error handling in completion handlers
  3. User Experience: Provide visual feedback to the user while the device is emitting sound
  4. Stop When Found: Always provide a way for users to stop the find operation

Notes​

  • The find device feature may not work if the device is powered off or out of Bluetooth range
  • The sound volume and pattern may vary depending on the device model
  • Some devices may have a limited duration for the find operation
