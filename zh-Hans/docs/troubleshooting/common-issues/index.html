Common Issues

This guide provides solutions to common issues you may encounter when using the AIBuds SDK.

SDK Initialization Issues​

SDK Not Initializing​

Symptom: The SDK fails to initialize with an error message.

Possible Causes:

  • Invalid app key
  • Network connectivity issues
  • Invalid bundle ID
  • SDK version incompatibility

Solutions:

  • Verify that you're using the correct app key
  • Check your network connection
  • Ensure your bundle ID matches the one registered with AIBuds
  • Update to the latest version of the SDK

Objective-C​

// Check for initialization errors
[AIBudsSDK initializeWithAppKey:@"YOUR_APP_KEY" completion:^(BOOL success, NSError *error) {
    if (success) {
        NSLog(@"AIBuds SDK initialized successfully");
    } else {
        NSLog(@"Failed to initialize AIBuds SDK: %@", error.localizedDescription);
        // Handle initialization error
    }
}];

Swift​

// Check for initialization errors
AIBudsSDK.initialize(withAppKey: "YOUR_APP_KEY") { success, error in
    if success {
        print("AIBuds SDK initialized successfully")
    } else {
        print("Failed to initialize AIBuds SDK: \(error?.localizedDescription ?? "Unknown error")")
        // Handle initialization error
    }
}

Device Connection Issues​

Device Not Found​

Symptom: The SDK cannot find your AIBuds device.

Possible Causes:

  • Bluetooth is disabled
  • Device is not in pairing mode
  • Device is out of range
  • Device battery is low

Solutions:

  • Enable Bluetooth on your iOS device
  • Put your AIBuds device in pairing mode
  • Move your device closer to your iOS device
  • Charge your AIBuds device

Objective-C​

// Start scanning for devices
[deviceManager startScanningForDevicesWithCompletion:^(AIBudsDevice *device) {
    NSLog(@"Found device: %@", device.name);
    // Connect to the device
    [deviceManager connectToDevice:device completion:^(BOOL success, NSError *error) {
        if (success) {
            NSLog(@"Connected to device: %@", device.name);
        } else {
            NSLog(@"Failed to connect to device: %@", error.localizedDescription);
        }
    }];
}];

// Stop scanning after 30 seconds
dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(30.0 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{    [deviceManager stopScanningForDevices];
    NSLog(@"Scanning stopped");
});

Swift​

// Start scanning for devices
deviceManager.startScanningForDevices { device in
    print("Found device: \(device.name)")
    // Connect to the device
    deviceManager.connect(to: device) { success, error in
        if success {
            print("Connected to device: \(device.name)")
        } else {
            print("Failed to connect to device: \(error?.localizedDescription ?? "Unknown error")")
        }
    }
}

// Stop scanning after 30 seconds
DispatchQueue.main.asyncAfter(deadline: .now() + 30) {
    deviceManager.stopScanningForDevices()
    print("Scanning stopped")
}

Connection Dropping​

Symptom: The connection to your AIBuds device keeps dropping.

Possible Causes:

  • Bluetooth interference
  • Device is out of range
  • Low battery
  • Firmware issues

Solutions:

  • Move away from sources of Bluetooth interference (microwaves, Wi-Fi routers, etc.)
  • Keep your iOS device and AIBuds device within range
  • Charge your AIBuds device
  • Update your AIBuds device firmware

Audio Issues​

No Audio Output​

Symptom: No sound is coming from your AIBuds device.

Possible Causes:

  • Device is muted
  • Volume is too low
  • Audio source is not playing
  • Connection issues

Solutions:

  • Unmute your device
  • Increase the volume
  • Ensure your audio source is playing
  • Reconnect your device

Objective-C​

// Check volume and unmute
[audioManager setVolume:0.8 completion:^(BOOL success, NSError *error) {
    if (success) {
        NSLog(@"Volume set to 80%");
    }
}];

// Check audio stream status
if (!audioStream.isPlaying) {
    [audioStream startWithCompletion:^(BOOL success, NSError *error) {
        if (success) {
            NSLog(@"Audio stream started");
        }
    }];
}

Swift​

// Check volume and unmute
audioManager.setVolume(0.8) { success, error in
    if success {
        print("Volume set to 80%")
    }
}

// Check audio stream status
if !audioStream.isPlaying {
    audioStream.start { success, error in
        if success {
            print("Audio stream started")
        }
    }
}

Poor Audio Quality​

Symptom: The audio quality from your AIBuds device is poor.

Possible Causes:

  • Low bitrate
  • Poor connection
  • Equalizer settings
  • Environmental noise

Solutions:

  • Increase the audio bitrate
  • Improve the Bluetooth connection
  • Adjust equalizer settings
  • Use noise cancellation in noisy environments

AI Feature Issues​

AI Features Not Working​

Symptom: AI features like smart noise cancellation or voice recognition are not working.

Possible Causes:

  • AI features are disabled
  • Network connectivity issues
  • Low battery
  • Firmware issues

Solutions:

  • Enable AI features
  • Check your network connection
  • Charge your device
  • Update your device firmware

Objective-C​

// Enable AI features
[aiManager setAIFeaturesEnabled:YES forDevice:device completion:^(BOOL success, NSError *error) {
    if (success) {
        NSLog(@"AI features enabled");
    } else {
        NSLog(@"Failed to enable AI features: %@", error.localizedDescription);
    }
}];

// Check AI features status
BOOL aiEnabled = [aiManager getAIFeaturesEnabledForDevice:device];
NSLog(@"AI features enabled: %@", aiEnabled ? @"Yes" : @"No");

Swift​

// Enable AI features
aiManager.setAIFeaturesEnabled(true, for: device) { success, error in
    if success {
        print("AI features enabled")
    } else {
        print("Failed to enable AI features: \(error?.localizedDescription ?? "Unknown error")")
    }
}

// Check AI features status
let aiEnabled = aiManager.getAIFeaturesEnabled(for: device)
print("AI features enabled: \(aiEnabled ? "Yes" : "No")")

Voice Recognition Issues​

Symptom: Voice recognition is not working correctly.

Possible Causes:

  • Voice recognition is disabled
  • Background noise
  • Poor microphone quality
  • Incorrect language settings

Solutions:

  • Enable voice recognition
  • Reduce background noise
  • Ensure the microphone is not blocked
  • Set the correct language

Battery Issues​

Rapid Battery Drain​

Symptom: Your AIBuds device battery is draining quickly.

Possible Causes:

  • AI features are enabled
  • High noise cancellation level
  • Continuous audio streaming
  • Background app refresh

Solutions:

  • Disable unnecessary AI features
  • Use lower noise cancellation levels when possible
  • Reduce audio streaming time
  • Disable background app refresh for your app

Objective-C​

// Optimize battery usage
- (void)optimizeBatteryUsage {
    // Disable AI features when not needed
    [aiManager setAIFeaturesEnabled:NO forDevice:device completion:^(BOOL success, NSError *error) {
        if (success) {
            NSLog(@"AI features disabled to save battery");
        }
    }];
    
    // Use lower noise cancellation level
    [audioManager setNoiseCancellationLevel:AIBudsNoiseCancellationLevelLow forDevice:device completion:^(BOOL success, NSError *error) {
        if (success) {
            NSLog(@"Noise cancellation level set to low");
        }
    }];
}

Swift​

// Optimize battery usage
func optimizeBatteryUsage() {
    // Disable AI features when not needed
    aiManager.setAIFeaturesEnabled(false, for: device) { success, error in
        if success {
            print("AI features disabled to save battery")
        }
    }
    
    // Use lower noise cancellation level
    audioManager.setNoiseCancellationLevel(.low, for: device) { success, error in
        if success {
            print("Noise cancellation level set to low")
        }
    }
}

Firmware Update Issues​

Firmware Update Failed​

Symptom: Firmware update for your AIBuds device failed.

Possible Causes:

  • Network connectivity issues
  • Low battery
  • Interrupted update
  • Incompatible firmware

Solutions:

  • Check your network connection
  • Ensure your device has sufficient battery
  • Try the update again
  • Contact AIBuds support for assistance

Objective-C​

// Check for firmware updates
[deviceManager checkForFirmwareUpdateForDevice:device completion:^(BOOL hasUpdate, NSString *version, NSString *releaseNotes, NSError *error) {
    if (hasUpdate) {
        NSLog(@"Firmware update available: %@", version);
        
        // Apply firmware update
        [deviceManager applyFirmwareUpdateForDevice:device progress:^(float progress) {
            NSLog(@"Update progress: %.2f%%", progress * 100);
        } completion:^(BOOL success, NSError *error) {
            if (success) {
                NSLog(@"Firmware update applied successfully");
            } else {
                NSLog(@"Failed to apply firmware update: %@", error.localizedDescription);
                // Handle update failure
            }
        }];
    } else {
        NSLog(@"No firmware update available");
    }
}];

Swift​

// Check for firmware updates
deviceManager.checkForFirmwareUpdate(for: device) { hasUpdate, version, releaseNotes, error in
    if hasUpdate {
        print("Firmware update available: \(version ?? "Unknown version")")
        
        // Apply firmware update
        deviceManager.applyFirmwareUpdate(for: device) { progress in
            print("Update progress: \(progress * 100)%")
        } completion: { success, error in
            if success {
                print("Firmware update applied successfully")
            } else {
                print("Failed to apply firmware update: \(error?.localizedDescription ?? "Unknown error")")
                // Handle update failure
            }
        }
    } else {
        print("No firmware update available")
    }
}

App Crashes​

App Crashing When Using SDK​

Symptom: Your app crashes when using the AIBuds SDK.

Possible Causes:

  • SDK version incompatibility
  • Memory issues
  • Null pointer exceptions
  • Threading issues

Solutions:

  • Update to the latest version of the SDK
  • Check for memory leaks
  • Add null checks for all SDK calls
  • Ensure SDK calls are made on the main thread

Objective-C​

// Ensure SDK calls are made on the main thread
dispatch_async(dispatch_get_main_queue(), ^{    // Make SDK calls here
    [deviceManager connectToDevice:device completion:^(BOOL success, NSError *error) {
        if (success) {
            NSLog(@"Connected to device: %@", device.name);
        } else {
            NSLog(@"Failed to connect to device: %@", error.localizedDescription);
        }
    }];
});

Swift​

// Ensure SDK calls are made on the main thread
DispatchQueue.main.async {
    // Make SDK calls here
    deviceManager.connect(to: device) { success, error in
        if success {
            print("Connected to device: \(device.name)")
        } else {
            print("Failed to connect to device: \(error?.localizedDescription ?? "Unknown error")")
        }
    }
}

Network Issues​

Network Timeout​

Symptom: SDK operations are timing out due to network issues.

Possible Causes:

  • Poor network connection
  • Server issues
  • Firewall restrictions
  • VPN issues

Solutions:

  • Check your network connection
  • Try again later if it's a server issue
  • Check firewall settings
  • Disable VPN if it's causing issues

Objective-C​

// Set network timeout
[thirdPartyManager configureStarburstSdkWithConfig:@{@"timeout": @(60.0)}];

// Handle network errors
[aiManager startAIChatWithPrompt:@"What's the weather today?" forDevice:device completion:^(NSString *response, NSError *error) {
    if (response) {
        NSLog(@"AI response: %@", response);
    } else {
        NSLog(@"Failed to get AI response: %@", error.localizedDescription);
        // Handle network error
        if ([error.domain isEqualToString:NSURLErrorDomain] && error.code == NSURLErrorTimedOut) {
            NSLog(@"Network timeout. Please check your connection.");
        }
    }
}];

Swift​

// Set network timeout
thirdPartyManager.configureStarburstSdk(withConfig: ["timeout": 60.0])

// Handle network errors
aiManager.startAIChat(withPrompt: "What's the weather today?", for: device) { response, error in
    if let response = response {
        print("AI response: \(response)")
    } else {
        print("Failed to get AI response: \(error?.localizedDescription ?? "Unknown error")")
        // Handle network error
        if let error = error as NSError?, error.domain == NSURLErrorDomain, error.code == NSURLErrorTimedOut {
            print("Network timeout. Please check your connection.")
        }
    }
}

Debugging Tips​

Enable Debug Logging​

Symptom: You need more information about what's happening in the SDK.

Solution: Enable debug logging to get more detailed information.

Objective-C​

// Enable debug logging
[AIBudsLog setLogLevel:AIBudsLogLevelDebug];

// Enable console logging
[AIBudsLogManager.shared setConsoleLoggingEnabled:YES];

// Enable file logging
[AIBudsLogManager.shared setFileLoggingEnabled:YES];

Swift​

// Enable debug logging
AIBudsLog.setLogLevel(.debug)

// Enable console logging
AIBudsLogManager.shared.setConsoleLoggingEnabled(true)

// Enable file logging
AIBudsLogManager.shared.setFileLoggingEnabled(true)

Export Logs​

Symptom: You need to share logs with AIBuds support for debugging.

Solution: Export logs from the SDK.

Objective-C​

// Export logs
[AIBudsLogManager.shared exportLogsWithCompletion:^(NSURL *fileURL, NSError *error) {
    if (fileURL) {
        NSLog(@"Logs exported to: %@", fileURL.path);
        
        // Share the log file
        UIActivityViewController *activityVC = [[UIActivityViewController alloc] initWithActivityItems:@[fileURL] applicationActivities:nil];
        [self presentViewController:activityVC animated:YES completion:nil];
    } else {
        NSLog(@"Failed to export logs: %@", error.localizedDescription);
    }
}];

Swift​

// Export logs
AIBudsLogManager.shared.exportLogs { fileURL, error in
    if let fileURL = fileURL {
        print("Logs exported to: \(fileURL.path)")
        
        // Share the log file
        let activityVC = UIActivityViewController(activityItems: [fileURL], applicationActivities: nil)
        self.present(activityVC, animated: true, completion: nil)
    } else {
        print("Failed to export logs: \(error?.localizedDescription ?? "Unknown error")")
    }
}

Contact Support​

If you're still experiencing issues with the AIBuds SDK, please contact AIBuds support:

  • Email: support@aibuds.com
  • Website: https://www.aibuds.com/support
  • Phone: +1 (800) 123-4567

When contacting support, please include:

  • Your app's bundle ID
  • The version of the AIBuds SDK you're using
  • The model of your AIBuds device
  • The firmware version of your AIBuds device
  • Steps to reproduce the issue
  • Any error messages you're seeing
  • Logs from the SDK
