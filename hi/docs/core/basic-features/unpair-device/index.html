डिवाइस अनपेयर करें

डिवाइस अनपेयर ऑपरेशन डिवाइस और कनेक्टेड डिवाइस के बीच पेयरिंग रिलेशनशिप को हटा देता है। यह ऑपरेशन तब उपयोगी होता है जब आपको डिवाइस को स्थायी रूप से डिस्कनेक्ट करने की आवश्यकता होती है या किसी अन्य डिवाइस के साथ पेयरिंग की तैयारी करनी होती है।

पूर्वापेक्षाएं​

अनपेयर करने से पहले, सुनिश्चित करें:

  • डिवाइस कनेक्ट है और स्थिर स्थिति में है
  • कोई भी आवश्यक डेटा सिंक्रनाइज़ेशन पूर्ण हो गया है
  • उपयोगकर्ता समझता है कि अनपेयरिंग के बाद डिवाइस डिस्कनेक्ट हो जाएगा

API संदर्भ​

फ्रेमवर्क​

AIBuds.xcframework

प्रोटोकॉल​

unpair मेथड निम्नलिखित प्रोटोकॉल में परिभाषित है। प्रोटोकॉल बेस डिवाइस API प्रोटोकॉल से इनहेरिट करता है।

  • Swift
  • Objective-C
/// अनपेयर सहित सामान्य डिवाइस ऑपरेशंस को परिभाषित करता है
protocol DeviceCommonAPI: DeviceAPI {
    /// डिवाइस का पेयरिंग हटाएं
    /// - Parameters:
    ///   - completion: ऑपरेशन रिज़ल्ट लौटाने वाला कम्प्लीशन कॉलबैक
    ///     - success: ऑपरेशन सफल होने पर `true`; अन्यथा `false`।
    ///     - error: एक `NSError` ऑब्जेक्ट जो हुई त्रुटि का वर्णन करता है, या `nil` यदि ऑपरेशन सफल रहा।
    func unpair(_ completion: AIBudsCompletionHandler?)
}
/// अनपेयर सहित सामान्य डिवाइस ऑपरेशंस को परिभाषित करता है
@protocol AIBudsDeviceCommonAPI <AIBudsDeviceAPI>
    /// डिवाइस का पेयरिंग हटाएं
    /// - Parameters:
    ///   - completion: ऑपरेशन रिज़ल्ट लौटाने वाला कम्प्लीशन कॉलबैक
    ///     - success: ऑपरेशन सफल होने पर `true`; अन्यथा `false`।
    ///     - error: एक `NSError` ऑब्जेक्ट जो हुई त्रुटि का वर्णन करता है, या `nil` यदि ऑपरेशन सफल रहा।
    - (void)unpairWithCompletion:(AIBudsCompletionHandler)completion;
@end

पैरामीटर​

पैरामीटरप्रकारविवरण
completionAIBudsCompletionHandler?वैकल्पिक कम्प्लीशन कॉलबैक जो ऑपरेशन पूरा होने पर कॉल किया जाता है।

कॉलबैक पैरामीटर:

नामप्रकारविवरण
successBoolऑपरेशन सफल होने पर true, अन्यथा false
errorNSError?त्रुटि होने पर त्रुटि जानकारी होती है, अन्यथा nil

रिटर्न वैल्यू​

यह मेथड सीधे कोई वैल्यू रिटर्न नहीं करता। रिज़ल्ट कम्प्लीशन कॉलबैक के माध्यम से प्रदान किया जाता है।

उपयोग उदाहरण​

  • Swift
  • Objective-C
import AIBuds

class DeviceManager {

    /// कनेक्टेड डिवाइस
    weak var device: DeviceConvertible?

    /// कनेक्टेड डिवाइस का पेयरिंग हटाता है
    func unpairDevice() {
        // सुनिश्चित करें कि डिवाइस अनपेयर प्रोटोकॉल सपोर्ट करता है
        guard let device = device as? DeviceCommonAPI else {
            print("डिवाइस अनपेयर सपोर्ट नहीं करता")
            return
        }

        // कम्प्लीशन हैंडलर के साथ अनपेयर एक्जीक्यूट करें
        device.unpair { [weak self] success, error in
            // फेलियर केस हैंडल करें
            if !success {
                let errorMessage = {
                    if let error = error {
                        return "\(error)"
                    }
                    return "अज्ञात त्रुटि"
                }()
                print("अनपेयर विफल: \(errorMessage)")
                return
            }
            // सक्सेस केस हैंडल करें
            print("अनपेयर सफलतापूर्वक पूर्ण")
        }
    }
}
#import <AIBuds/AIBuds.h>

@interface DeviceManager ()

/// कनेक्टेड डिवाइस
@property (weak, nonatomic) id<AIBudsDeviceConvertible> device;

@end

@implementation DeviceManager

- (void)unpairDevice {
    __weak typeof(self) weakSelf = self;


    id<AIBudsDeviceCommonAPI> device = (id<AIBudsDeviceCommonAPI>)self.device;
    // सुनिश्चित करें कि डिवाइस अनपेयर प्रोटोकॉल सपोर्ट करता है
    if ([device conformsToProtocol:@protocol(AIBudsDeviceCommonAPI)]) {
        // कम्प्लीशन हैंडलर के साथ अनपेयर एक्जीक्यूट करें
        [device unpairWithCompletion:^(BOOL success, NSError * _Nullable error) {
            // फेलियर केस हैंडल करें
            if (!success) {
                NSLog(@"अनपेयर विफल: %@", error);
                return;
            }
            // सक्सेस केस हैंडल करें
            NSLog(@"अनपेयर सफलतापूर्वक पूर्ण");
        }];
    }
}

@end

सर्वोत्तम अभ्यास​

  1. उपयोगकर्ता से पुष्टि करें: अनपेयर शुरू करने से पहले हमेशा एक पुष्टि संवाद प्रदर्शित करें, क्योंकि यह ऑपरेशन डिवाइस को डिस्कनेक्ट कर देगा और पुनः पेयरिंग की आवश्यकता होगी।

  2. बैकग्राउंड एक्जीक्यूशन को हैंडल करें: UI अपडेट करते समय कम्प्लीशन हैंडलर को DispatchQueue.main.async ब्लॉक में रैप करें।

  3. वीक सेल्फ रेफरेंस: रिटेन साइकिल से बचने के लिए कम्प्लीशन हैंडलर में [weak self] का उपयोग करें।

  4. प्रोटोकॉल कंफर्मेंस की जांच करें: मेथड को कॉल करने से पहले सत्यापित करें कि डिवाइस DeviceCommonAPI प्रोटोकॉल का अनुपालन करता है।

  5. डिस्कनेक्शन को हैंडल करें: सफल अनपेयर के बाद, डिवाइस डिस्कनेक्शन को सुव्यवस्थित रूप से हैंडल करें और पुनः पेयरिंग के लिए मार्गदर्शन प्रदान करें।

प्लेटफॉर्म सीमाएं​

iOS सिस्टम ब्लूटूथ सीमा​

iOS पर, सिस्टम ब्लूटूथ सेटिंग्स एप्लिकेशन को प्रोग्रामेटिक रूप से ब्लूटूथ डिवाइस का पेयरिंग हटाने की अनुमति नहीं देती हैं। यह Apple द्वारा सुरक्षा और उपयोगकर्ता नियंत्रण के लिए लगाई गई सिस्टम-स्तरीय प्रतिबंध है।

इसका मतलब:

  • unpair कॉल करने के बाद भी, iOS डिवाइस के लिए BLE बॉन्डिंग जानकारी बनाए रखेगा
  • ऐप रीस्टार्ट या ब्लूटूथ सक्षम होने पर डिवाइस स्वचालित रूप से पुनः कनेक्ट हो सकता है
  • डिवाइस iOS सेटिंग्स > ब्लूटूथ में दिखाई देता रहेगा

अनुशंसित उपयोगकर्ता मार्गदर्शन:

अपने iOS ऐप में अनपेयर कार्यक्षमता लागू करते समय, आपको उपयोगकर्ताओं को iOS सेटिंग्स से मैन्युअल रूप से अनपेयर करने के लिए मार्गदर्शन करना चाहिए:

  1. पूर्ण अनपेयर के लिए: उपयोगकर्ताओं को सेटिंग्स > ब्लूटूथ पर जाने, डिवाइस खोजने, "i" आइकन टैप करने और "इस डिवाइस को भूलें" चुनने के लिए कहें
  2. स्पष्ट UI फीडबैक प्रदान करें: जब उपयोगकर्ता अनपेयर का अनुरोध करे तो निर्देश या ब्लूटूथ सेटिंग्स के लिए डीप लिंक दिखाएं
  3. ऐप-स्तरीय डिस्कनेक्शन: unpair मेथड अभी भी आपके ऐप के दृष्टिकोण से डिवाइस को डिस्कनेक्ट करेगा, लेकिन सिस्टम बॉन्डिंग बनी रहेगी
  • Swift
  • Objective-C
/// उपयोगकर्ता को iOS सेटिंग्स से अनपेयर करने के लिए प्रेरित करता है
func promptUserToUnpairFromSettings() {
    // निर्देशों के साथ अलर्ट दिखाएं
    let alert = UIAlertController(
        title: "डिवाइस अनपेयर करें",
        message: "डिवाइस को पूरी तरह से अनपेयर करने के लिए, कृपया सेटिंग्स > ब्लूटूथ पर जाएं, अपने डिवाइस के बगल में 'i' आइकन टैप करें, और 'इस डिवाइस को भूलें' चुनें।",
        preferredStyle: .alert
    )
    alert.addAction(UIAlertAction(title: "सेटिंग्स खोलें", style: .default) { _ in
        // ब्लूटूथ सेटिंग्स के लिए डीप लिंक
        if let url = URL(string: "App-prefs:Bluetooth") {
            UIApplication.shared.open(url)
        }
    })
    alert.addAction(UIAlertAction(title: "रद्द करें", style: .cancel))
    
    // अलर्ट प्रस्तुत करें
    if let viewController = UIApplication.shared.windows.first?.rootViewController {
        viewController.present(alert, animated: true)
    }
}
/// उपयोगकर्ता को iOS सेटिंग्स से अनपेयर करने के लिए प्रेरित करता है
- (void)promptUserToUnpairFromSettings {
    UIAlertController *alert = [UIAlertController 
        alertControllerWithTitle:@"डिवाइस अनपेयर करें"
        message:@"डिवाइस को पूरी तरह से अनपेयर करने के लिए, कृपया सेटिंग्स > ब्लूटूथ पर जाएं, अपने डिवाइस के बगल में 'i' आइकन टैप करें, और 'इस डिवाइस को भूलें' चुनें।"
        preferredStyle:UIAlertControllerStyleAlert];
    
    [alert addAction:[UIAlertAction actionWithTitle:@"सेटिंग्स खोलें" style:UIAlertActionStyleDefault handler:^(UIAlertAction * _Nonnull action) {
        NSURL *url = [NSURL URLWithString:@"App-prefs:Bluetooth"];
        if (url && [[UIApplication sharedApplication] canOpenURL:url]) {
            [[UIApplication sharedApplication] openURL:url options:@{} completionHandler:nil];
        }
    }]];
    
    [alert addAction:[UIAlertAction actionWithTitle:@"रद्द करें" style:UIAlertActionStyleCancel handler:nil]];
    
    UIViewController *viewController = [UIApplication sharedApplication].windows.firstObject.rootViewController;
    [viewController presentViewController:alert animated:YES completion:nil];
}

  • अनपेयर होने में कुछ सेकंड लग सकते हैं
