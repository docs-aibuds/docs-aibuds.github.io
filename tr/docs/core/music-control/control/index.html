Music Control

Control music playback operations such as play, pause, next track, previous track, and volume adjustment.

Prerequisites​

Before calling this API, ensure:

  • The device is connected and in a stable state
  • The device supports music control functionality

API Reference​

Framework​

AIBuds.xcframework

Import​

  • Swift
  • Objective-C
import AIBuds
#import <AIBuds/AIBuds.h>
#import <AIBuds/AIBuds-Swift.h>

Protocol​

The music control methods are defined in the following protocol:

  • Swift
  • Objective-C
protocol MusicControlAPI: DeviceAPI {
    func play(completion: AIBudsCompletionHandler?)
    func pause(completion: AIBudsCompletionHandler?)
    func nextTrack(completion: AIBudsCompletionHandler?)
    func previousTrack(completion: AIBudsCompletionHandler?)
}
@protocol AIBudsMusicControlAPI <AIBudsDeviceAPI>
- (void)playWithCompletion:(AIBudsCompletionHandler)completion;
- (void)pauseWithCompletion:(AIBudsCompletionHandler)completion;
- (void)nextTrackWithCompletion:(AIBudsCompletionHandler)completion;
- (void)previousTrackWithCompletion:(AIBudsCompletionHandler)completion;
@end

Parameters​

ParameterTypeDescription
completionAIBudsCompletionHandler?Optional completion callback that is called when the operation completes.

Callback Parameters:

NameTypeDescription
successBooltrue if the operation succeeded, false otherwise.
errorNSError?Contains error information if the operation failed, nil otherwise.

Usage Examples​

  • Swift
  • Objective-C
import AIBuds

class MusicManager {
    weak var device: DeviceConvertible?
    
    func controlMusic(action: MusicAction) {
        guard let device = device as? MusicControlAPI else {
            print("Device does not support MusicControlAPI")
            return
        }
        
        switch action {
        case .play:
            device.play { success, error in
                if let error = error {
                    print("Failed to play: \(error)")
                }
            }
        case .pause:
            device.pause { success, error in
                if let error = error {
                    print("Failed to pause: \(error)")
                }
            }
        case .next:
            device.nextTrack { success, error in
                if let error = error {
                    print("Failed to skip to next track: \(error)")
                }
            }
        case .previous:
            device.previousTrack { success, error in
                if let error = error {
                    print("Failed to skip to previous track: \(error)")
                }
            }
        }
    }
}

enum MusicAction {
    case play
    case pause
    case next
    case previous
}
#import <AIBuds/AIBuds.h>

typedef NS_ENUM(NSInteger, MusicAction) {
    MusicActionPlay,
    MusicActionPause,
    MusicActionNext,
    MusicActionPrevious
};

@interface MusicManager ()
@property (weak, nonatomic) id<AIBudsDeviceConvertible> device;
@end

@implementation MusicManager

- (void)controlMusicWithAction:(MusicAction)action {
    id<AIBudsMusicControlAPI> device = (id<AIBudsMusicControlAPI>)self.device;
    if (![device conformsToProtocol:@protocol(AIBudsMusicControlAPI)]) {
        NSLog(@"Device does not support MusicControlAPI");
        return;
    }
    
    switch (action) {
        case MusicActionPlay:
            [device playWithCompletion:^(BOOL success, NSError *error) {
                if (error) NSLog(@"Failed to play: %@", error);
            }];
            break;
        case MusicActionPause:
            [device pauseWithCompletion:^(BOOL success, NSError *error) {
                if (error) NSLog(@"Failed to pause: %@", error);
            }];
            break;
        case MusicActionNext:
            [device nextTrackWithCompletion:^(BOOL success, NSError *error) {
                if (error) NSLog(@"Failed to skip to next track: %@", error);
            }];
            break;
        case MusicActionPrevious:
            [device previousTrackWithCompletion:^(BOOL success, NSError *error) {
                if (error) NSLog(@"Failed to skip to previous track: %@", error);
            }];
            break;
    }
}

@end

Best Practices​

  1. Handle State: Track the current playback state to avoid unnecessary commands.
  2. Debounce Actions: Prevent rapid consecutive commands.

Notes​

  • Music control commands are sent to the connected audio device
  • Some devices may have hardware buttons that also control music playback
