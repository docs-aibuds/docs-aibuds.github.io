Get Work Mode

Retrieve the current work mode of the device.

Prerequisites​

Before calling this API, ensure:

  • The device is connected and in a stable state

API Reference​

Framework​

AIBuds.xcframework

Import​

  • Swift
  • Objective-C
import AIBuds
#import <AIBuds/AIBuds.h>
#import <AIBuds/AIBuds-Swift.h>

Protocol​

The getWorkMode method is defined in the following protocol:

  • Swift
  • Objective-C
protocol WorkModeAPI: DeviceAPI {
    func getWorkMode(_ completion: @escaping (Result<WorkMode, Error>) -> Void)
}

enum WorkMode {
    case normal
    case game
    case music
    case sports
    case custom(String)
}
@protocol AIBudsWorkModeAPI <AIBudsDeviceAPI>
- (void)getWorkModeWithCompletion:(void(^)(AIBudsWorkMode mode, NSError * _Nullable error))completion;
@end

typedef NS_ENUM(NSInteger, AIBudsWorkMode) {
    AIBudsWorkModeNormal,
    AIBudsWorkModeGame,
    AIBudsWorkModeMusic,
    AIBudsWorkModeSports,
    AIBudsWorkModeCustom
};

Parameters​

ParameterTypeDescription
completion(Result<WorkMode, Error>)Completion callback that returns the work mode or an error.

Usage Examples​

  • Swift
  • Objective-C
import AIBuds

class DeviceManager {
    weak var device: DeviceConvertible?
    
    func fetchWorkMode() {
        guard let device = device as? WorkModeAPI else {
            print("Device does not support WorkModeAPI")
            return
        }
        
        device.getWorkMode { result in
            switch result {
            case .success(let mode):
                print("Current work mode: \(mode)")
            case .failure(let error):
                print("Failed to get work mode: \(error)")
            }
        }
    }
}
#import <AIBuds/AIBuds.h>

@interface DeviceManager ()
@property (weak, nonatomic) id<AIBudsDeviceConvertible> device;
@end

@implementation DeviceManager

- (void)fetchWorkMode {
    id<AIBudsWorkModeAPI> device = (id<AIBudsWorkModeAPI>)self.device;
    if ([device conformsToProtocol:@protocol(AIBudsWorkModeAPI)]) {
        [device getWorkModeWithCompletion:^(AIBudsWorkMode mode, NSError *error) {
            if (error) {
                NSLog(@"Failed to get work mode: %@", error);
                return;
            }
            NSLog(@"Current work mode: %ld", (long)mode);
        }];
    }
}

@end

Error Handling​

Error Domain: AIBudsSDK.ErrorDomain

Error CodeDescriptionRecovery Suggestion
.deviceNotConnectedDevice is not connectedEnsure device is paired and connected
.bleCommandExecFailedDueToTimeoutOperation timed outRetry the operation
.deviceNotSupportWork mode query is not supportedCheck device capabilities

Best Practices​

  1. Listen for Changes: Register for work mode change notifications.
  2. Update UI: Refresh UI when work mode changes.

Notes​

  • Available work modes vary by device model
  • Some devices may only support normal and game modes
